# 短故事项目

存放3-5万字的独立短故事，每个故事一个子目录。

## 目录结构

```
/opt/data/novel/短故事/
├── settings/              # 通用设定模板/写作风格规则
├── chapters/              # 正文（按故事名分子目录）
│   ├── {故事A}/
│   │   ├── story_001.txt
│   │   └── ...
│   └── {故事B}/
│       └── story_001.txt
└── README.md              # 本文件
```

## 短故事 vs 长篇的区别

- **篇幅**：3-5万字（10-15章，每章3000字左右）
- **节奏**：高密度，不注水，每章都有信息增量
- **验证快**：写完即可发布，快速看数据
- **改编潜力**：短剧改编成功率40%+，番茄奖励最高300万/部
- **设定轻量**：不需要70章分章纲，只需要故事大纲+人物表+伏笔表

## 写作流程

1. 确定故事题材和核心悬念（1句话能说清）
2. settings/{故事名}/ 下生成：故事大纲.md、人物表.md、伏笔表.md
3. 单章生成（复用 web-novel-writing 技能的生成流程）
4. 审稿→发布
